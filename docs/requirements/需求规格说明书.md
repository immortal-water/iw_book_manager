## 1. 引言

### 1.1 编写目的

本文档旨在明确“个人图书管理助手”系统的功能需求和非功能需求，为系统设计、开发实现、测试验收及后续维护提供统一、准确的依据。

### 1.2 项目背景

随着个人藏书和电子资料不断增多，如何高效管理图书信息成为实际问题。现有的通用笔记或表格工具缺乏针对图书管理专门优化，无法满足分类检索、状态跟踪等需求。本项目旨在开发一个轻量级的个人图书管理助手，帮助用户便捷地记录、查找和整理图书。

### 1.3 术语与缩写

| 术语 | 说明 |
|------|------|
| 图书 | 用户录入的纸质或电子书籍信息条目 |
| 分类 | 图书的类别归属，如 Java、Python、数据库等 |
| 阅读状态 | 标识图书阅读进度：未读、在读、已读 |
| CRUD | 增(Create)、删(Delete)、改(Update)、查(Read)基础操作 |

## 2. 系统概述

### 2.1 系统目标

- 实现图书信息的增删改查功能，管理个人藏书
- 支持按书名、作者进行模糊搜索，快速定位图书
- 支持分类管理，可按分类维度统计数据
- 展示阅读状态，跟踪阅读进度
- 提供直观的统计看板，展示总数和各类别数量

### 2.2 用户角色

| 角色 | 说明 |
|------|------|
| 普通用户 | 系统的唯一使用者，拥有全部图书管理操作权限 |

## 3. 功能需求

### 3.1 图书管理模块

#### 3.1.1 添加图书

- **描述：** 用户录入新图书信息并保存到数据库
- **输入：** 书名（必填）、作者（必填）、ISBN（选填）、分类（下拉选择）、位置（选填）、状态（默认“未读”）
- **处理流程：**
  1. 用户点击“+ 添加图书”按钮，弹出空白表单
  2. 用户填写图书信息
  3. 前端检查书名、作者非空
  4. 通过 AJAX 将表单数据提交至 `POST /book/add`
  5. 服务器使用参数化 SQL 插入 book 表
  6. 返回成功状态，前端刷新页面
- **异常处理：** 必填项为空时浏览器阻止提交；数据库错误时返回失败提示

#### 3.1.2 图书列表展示

- **描述：** 以表格形式展示所有图书信息
- **显示字段：** ID、书名、作者、分类名称、位置、状态、操作按钮（编辑/删除）
- **数据来源：** book 表与 category 表联表查询
- **附加功能：** 表格上方显示统计看板和搜索框

#### 3.1.3 编辑图书

- **描述：** 修改已保存的图书信息
- **操作流程：**
  1. 用户在列表中点击某图书的“编辑”按钮
  2. 前端请求 `GET /book/get/<id>` 获取当前数据
  3. 弹窗预填充原有信息
  4. 用户修改需要更新的字段
  5. 点击“保存”，提交至 `POST /book/update/<id>`
  6. 数据库更新，页面刷新
- **可修改字段：** 所有字段均可修改

#### 3.1.4 删除图书

- **描述：** 从系统中移除图书记录
- **操作流程：**
  1. 用户点击“删除”按钮
  2. 弹出确认对话框：“确定删除吗？”
  3. 用户点击“确定”，发送 `GET /book/delete/<id>`
  4. 数据库删除对应记录
  5. 页面刷新
- **约束：** 删除操作不可逆；用户点击“取消”则不执行任何操作

### 3.2 搜索功能

- **描述：** 根据关键字匹配书名或作者字段
- **匹配方式：** 模糊匹配（SQL `LIKE '%关键词%'`）
- **输入：** 任意文本关键词
- **输出：** 匹配到的图书列表；若无匹配则显示空表格
- **操作方式：** 搜索框输入关键词后点击“搜索”按钮或按回车键

### 3.3 统计看板

- **描述：** 在页面上方动态展示图书汇总数据
- **数据项：** 总图书数、每个分类的图书数量
- **更新方式：** 页面加载时通过 Fetch API 异步请求 `/stats`，获取 JSON 数据后渲染为卡片

### 3.4 分类管理

- **描述：** 系统内置 5 个分类：Java、Python、数据库、前端、其他
- **使用方式：** 添加或编辑图书时从下拉列表中选择
- **约束：** 当前版本不支持用户动态增删分类

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 2 秒 |
| 搜索响应时间 | < 1 秒（不超过 500 条数据） |
| 支持数据量 | ≥ 500 本图书记录 |

### 4.2 安全性需求

- 所有数据库操作均使用参数化查询（`?` 占位符），防止 SQL 注入
- 系统仅在本地环回地址（127.0.0.1）监听，无外部网络暴露
- 前端对书名、作者必填项进行校验

### 4.3 兼容性需求

- 浏览器：Chrome 最新版、Edge 最新版
- 操作系统：Windows 10/11

### 4.4 可维护性需求

- 数据库建表脚本独立存放于 `sql/schema.sql`
- 核心代码使用注释说明
- Git 提交记录规范，采用 `<type>: <描述> - by <姓名>` 格式
- 首次运行自动调用建表脚本，无需手动初始化

## 5. 数据需求

### 5.1 数据实体关系

- 分类（Category）与图书（Book）为一对多关系
- 一个分类可以包含多本图书，一本图书只属于一个分类

### 5.2 图书表（book）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | 主键，自增 | 图书记录唯一标识 |
| title | TEXT | 非空 | 书名 |
| author | TEXT | 非空 | 作者 |
| isbn | TEXT | 可选 | 国际标准书号 |
| category_id | INTEGER | 外键，关联 category.id | 所属分类 |
| location | TEXT | 可选 | 存放位置 |
| status | TEXT | 默认“未读” | 阅读状态：未读 / 在读 / 已读 |
| created_at | TIMESTAMP | 默认当前时间 | 创建时间 |

### 5.3 分类表（category）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | 主键，自增 | 分类标识 |
| name | TEXT | 唯一，非空 | 分类名称 |

## 6. 接口需求

| 路由 | 方法 | 功能 | 返回类型 |
|------|------|------|----------|
| `/` | GET | 首页，展示图书列表及统计 | HTML 页面 |
| `/book/add` | POST | 添加图书 | JSON |
| `/book/get/<id>` | GET | 获取单本图书信息 | JSON |
| `/book/update/<id>` | POST | 更新图书信息 | JSON |
| `/book/delete/<id>` | GET | 删除指定图书 | JSON |
| `/stats` | GET | 获取统计数据 | JSON |

---

## 7. 业务流程

系统主要业务流程如下：

1. 用户访问首页，系统加载图书列表和统计看板
2. 用户可通过搜索框输入关键词筛选图书
3. 用户点击“添加图书”弹出表单，填写信息后保存
4. 用户点击“编辑”修改已有图书信息
5. 用户点击“删除”并确认后移除图书
6. 统计看板在每次页面加载时自动更新
