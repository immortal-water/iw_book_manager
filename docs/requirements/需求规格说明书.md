## 1. 引言

### 1.1 编写目的

本文档旨在明确"个人图书管理助手"系统的全部功能需求和非功能需求，为系统设计、开发实现、测试验收及后续维护提供统一、准确的依据。本版本覆盖核心CRUD功能及6个扩展功能。

### 1.2 项目背景

随着个人藏书和电子资料不断增多，如何高效管理图书信息成为实际问题。现有的通用笔记或表格工具缺乏针对图书管理专门优化，无法满足分类检索、状态跟踪等需求。本项目旨在开发一个轻量级的个人图书管理助手，帮助用户便捷地记录、查找和整理图书。

### 1.3 术语与缩写

| 术语 | 说明 |
|------|------|
| 图书 | 用户录入的纸质或电子书籍信息条目 |
| 分类 | 图书的类别归属，如 Java、Python、数据库等 |
| 标签 | 图书的自定义标记，支持多对多关联，如"经典""入门" |
| 阅读状态 | 标识图书阅读进度：未读、在读、已读 |
| CRUD | 增(Create)、删(Delete)、改(Update)、查(Read)基础操作 |
| CSV | 逗号分隔值文件格式，可用于 Excel 打开 |

## 2. 系统概述

### 2.1 系统目标

- 实现图书信息的增删改查功能，管理个人藏书
- 支持按书名、作者进行模糊搜索，快速定位图书
- 支持按阅读状态筛选图书
- 支持自定义标签系统，实现灵活的多维度分类
- 支持批量删除操作，提升管理效率
- 展示阅读状态和阅读进度条，跟踪阅读进度
- 提供直观的统计看板，展示总数、分类数量和状态分布
- 支持数据导出为CSV文件
- 支持分页浏览，适配大量图书场景

### 2.2 用户角色

| 角色 | 说明 |
|------|------|
| 普通用户 | 系统的唯一使用者，拥有全部图书管理操作权限 |

## 3. 功能需求

### 3.1 图书管理模块

#### 3.1.1 添加图书

- **描述：** 用户录入新图书信息并保存到数据库
- **输入：** 书名（必填）、作者（必填）、ISBN（选填）、分类（下拉选择）、位置（选填）、状态（默认"未读"）、标签（选填，逗号分隔）
- **处理流程：**
  1. 用户点击"+ 添加图书"按钮，弹出空白表单
  2. 用户填写图书信息
  3. 前端检查书名、作者非空
  4. 通过 AJAX 将表单数据提交至 `POST /book/add`
  5. 服务器使用参数化 SQL 插入 book 表
  6. 解析标签字段，对每个标签名查找已有标签或创建新标签
  7. 在 book_tag 表中建立图书与标签的关联
  8. 返回成功状态，前端刷新页面
- **异常处理：** 必填项为空时浏览器阻止提交；标签输入为空时跳过标签处理

#### 3.1.2 图书列表展示

- **描述：** 以表格形式分页展示所有图书信息
- **显示字段：** 复选框、ID、书名、作者、分类名称、标签、位置、状态徽章、迷你进度条、操作按钮（编辑/删除）
- **数据来源：** book 表与 category 表联表查询，关联查询 tag 表
- **每页条数：** 5条
- **排序规则：** 按 ID 倒序（最新添加的在前）
- **附加功能：** 表格上方显示统计看板、总体阅读进度条、搜索框、筛选控件、操作按钮

#### 3.1.3 编辑图书

- **描述：** 修改已保存的图书信息
- **操作流程：**
  1. 用户在列表中点击某图书的"编辑"按钮
  2. 前端请求 `GET /book/get/<id>` 获取当前数据（含标签列表）
  3. 弹窗预填充原有信息，标签自动转换为逗号分隔字符串回填
  4. 用户修改需要更新的字段
  5. 点击"保存"，提交至 `POST /book/update/<id>`
  6. 服务器更新 book 表
  7. 删除该图书所有旧标签关联
  8. 重新解析标签字段并建立新关联
  9. 页面刷新
- **可修改字段：** 所有字段均可修改

#### 3.1.4 删除图书

- **描述：** 从系统中移除单本图书记录
- **操作流程：**
  1. 用户点击"删除"按钮
  2. 弹出确认对话框："确定删除吗？"
  3. 用户点击"确定"，发送 `GET /book/delete/<id>`
  4. 数据库删除对应记录，book_tag 表通过外键级联自动清理
  5. 页面刷新
- **约束：** 删除操作不可逆

#### 3.1.5 批量删除图书

- **描述：** 勾选多本图书后一键删除
- **操作流程：**
  1. 用户勾选图书行的复选框
  2. 系统显示"🗑 批量删除(N)"按钮，N为选中数量
  3. 支持表头全选复选框一键选中/取消当前页所有图书
  4. 用户点击"批量删除"，弹出确认对话框
  5. 确认后前端通过 POST JSON 发送 ID 列表至 `/book/batch_delete`
  6. 后端使用 `WHERE id IN (...)` 批量删除，book_tag 级联清理
  7. 返回删除数量，页面刷新

### 3.2 搜索与筛选模块

#### 3.2.1 模糊搜索

- **描述：** 根据关键字匹配书名或作者字段
- **匹配方式：** 模糊匹配（SQL `LIKE '%关键词%'`）
- **输入：** 任意文本关键词
- **输出：** 匹配到的图书列表；若无匹配则显示"暂无图书数据"
- **特性：** 可与状态筛选和标签筛选组合使用

#### 3.2.2 按阅读状态筛选

- **描述：** 通过下拉框按阅读状态过滤图书
- **筛选选项：** 全部状态、未读、在读、已读
- **交互方式：** 选择后自动提交表单刷新页面
- **特性：** 下拉框保留当前选中值；可与搜索和标签筛选组合使用

#### 3.2.3 按标签筛选

- **描述：** 通过标签筛选栏按标签过滤图书
- **交互方式：** 点击标签名即筛选，当前选中标签蓝色高亮；点击"全部"清除筛选
- **数据来源：** tag 表中所有标签
- **筛选逻辑：** 通过子查询 `book.id IN (SELECT book_id FROM book_tag WHERE tag_id = ?)` 实现
- **特性：** 可与搜索和状态筛选组合使用，三条件同时生效

### 3.3 统计与展示模块

#### 3.3.1 统计看板

- **描述：** 在页面上方动态展示图书汇总数据
- **数据项：** 总图书数、每个分类的图书数量、未读数量、在读数量、已读数量
- **更新方式：** 页面加载时通过 Fetch API 异步请求 `/stats`，获取 JSON 数据后渲染为卡片

#### 3.3.2 阅读进度条

- **描述：** 可视化展示阅读进度
- **总体进度条：** 位于统计看板下方，根据全部图书状态计算整体进度百分比
- **颜色规则：** 红色（<30%）、橙色（30%-79%）、绿色（≥80%）
- **单本迷你进度条：** 每本图书行显示迷你进度条（未读=0%红色，在读=50%橙色，已读=100%绿色）
- **状态徽章：** 状态列以彩色徽章显示（未读红色、在读橙色、已读绿色）

### 3.4 数据管理模块

#### 3.4.1 数据导出CSV

- **描述：** 将图书列表导出为CSV文件下载
- **导出字段：** ID、书名、作者、ISBN、分类、位置、状态
- **特性：**
  - 导出全部数据，不受分页限制
  - 支持按当前搜索和状态筛选条件导出
  - 写入 UTF-8 BOM 头，确保 Excel 打开时中文正常显示
- **触发方式：** 点击"📥 导出 CSV"按钮

### 3.5 分类管理

- **描述：** 系统内置 5 个分类：Java、Python、数据库、前端、其他
- **使用方式：** 添加或编辑图书时从下拉列表中选择
- **约束：** 当前版本不支持用户动态增删分类

### 3.6 标签系统

- **描述：** 自定义标签，支持多对多关联
- **数据模型：** tag 表（id, name）+ book_tag 关联表（book_id, tag_id），联合主键，双向级联删除
- **预设标签：** 经典、入门、进阶、工具书、收藏（schema.sql 初始化）
- **标签创建：** 添加或编辑图书时输入不存在的标签名，系统自动创建
- **标签关联：** 使用 `INSERT OR IGNORE` 避免重复关联
- **标签清理：** 删除图书时 book_tag 关联记录级联删除；编辑图书时先删旧关联再建新关联
- **数据库迁移：** 提供 `migrate_db.py` 脚本，旧版数据库可无缝升级

### 3.7 分页功能

- **描述：** 图书列表分页展示
- **每页条数：** 5条
- **导航功能：** 首页、上一页、数字页码、下一页、末页
- **页码折叠：** 页码过多时中间用省略号折叠，首尾各保留3页，当前页前后各保留1页
- **当前页标识：** 绿色高亮
- **信息展示：** "共 X 条，第 Y/Z 页"
- **特性：** 分页与搜索、状态筛选、标签筛选联动

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 2 秒 |
| 搜索响应时间 | < 1 秒（不超过 500 条数据） |
| 支持数据量 | ≥ 500 本图书记录 |

### 4.2 安全性需求

- 所有数据库操作均使用参数化查询（`?` 占位符），防止 SQL 注入
- 系统仅在本地环回地址（127.0.0.1）监听，无外部网络暴露
- 前端对书名、作者必填项进行校验
- 批量删除接口使用 JSON 传递参数

### 4.3 兼容性需求

- 浏览器：Chrome 最新版、Edge 最新版
- 操作系统：Windows 10/11
- CSV 文件兼容 Microsoft Excel 打开

### 4.4 可维护性需求

- 数据库建表脚本独立存放于 `sql/schema.sql`
- 数据库迁移脚本 `migrate_db.py` 支持旧版升级
- 首次运行自动调用建表脚本，无需手动初始化
- 核心代码使用注释说明
- Git 提交记录规范，采用 `<type>: <描述> - by <姓名>` 格式

## 5. 数据需求

### 5.1 数据实体关系

- 分类（Category）与图书（Book）为一对多关系
- 图书（Book）与标签（Tag）为多对多关系，通过 book_tag 关联表实现

### 5.2 分类表（category）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | 主键，自增 | 分类标识 |
| name | TEXT | 唯一，非空 | 分类名称 |

### 5.3 图书表（book）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | 主键，自增 | 图书记录唯一标识 |
| title | TEXT | 非空 | 书名 |
| author | TEXT | 非空 | 作者 |
| isbn | TEXT | 可选 | 国际标准书号 |
| category_id | INTEGER | 外键，关联 category.id | 所属分类 |
| location | TEXT | 可选 | 存放位置 |
| status | TEXT | 默认"未读" | 阅读状态：未读 / 在读 / 已读 |
| created_at | TIMESTAMP | 默认当前时间 | 创建时间 |

### 5.4 标签表（tag）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | 主键，自增 | 标签标识 |
| name | TEXT | 唯一，非空 | 标签名称 |

### 5.5 图书标签关联表（book_tag）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| book_id | INTEGER | 外键，关联 book.id，级联删除 | 图书ID |
| tag_id | INTEGER | 外键，关联 tag.id，级联删除 | 标签ID |
| PRIMARY KEY | (book_id, tag_id) | 联合主键 | 防止重复关联 |

## 6. 接口需求

| 路由 | 方法 | 功能 | 请求参数 | 返回类型 |
|------|------|------|----------|----------|
| `/` | GET | 首页，展示图书列表及统计 | search, status, tag, page | HTML 页面 |
| `/book/add` | POST | 添加图书 | title, author, isbn, category_id, location, status, tags | JSON |
| `/book/get/<id>` | GET | 获取单本图书信息（含标签） | - | JSON |
| `/book/update/<id>` | POST | 更新图书信息及标签 | title, author, isbn, category_id, location, status, tags | JSON |
| `/book/delete/<id>` | GET | 删除指定图书 | - | JSON |
| `/book/batch_delete` | POST | 批量删除图书 | JSON: {ids: [...]} | JSON |
| `/stats` | GET | 获取统计数据（总数+分类+状态+进度） | - | JSON |
| `/export/csv` | GET | 导出图书为CSV文件 | search, status | CSV 文件 |

## 7. 业务流程

系统主要业务流程如下：

1. 用户访问首页，系统加载图书列表（分页）、统计看板和总体阅读进度条
2. 用户可通过搜索框按书名/作者模糊搜索
3. 用户可通过状态下拉框按阅读状态筛选
4. 用户可通过标签筛选栏按标签筛选
5. 搜索、状态筛选、标签筛选可任意组合使用
6. 用户点击"+ 添加图书"弹出表单，填写信息含标签后保存
7. 用户点击"编辑"修改已有图书信息，标签自动回填
8. 用户点击"删除"并确认后移除单本图书
9. 用户勾选多本图书后点击"批量删除"一键移除
10. 用户点击"导出 CSV"下载全部图书数据
11. 统计看板在每次页面加载时异步更新
