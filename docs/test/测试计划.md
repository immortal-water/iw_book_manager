## 1. 测试目标

验证"个人图书管理助手"系统全部已完成功能的正确性、数据准确性和界面可用性，覆盖核心CRUD功能及6个扩展功能（数据导出CSV、批量删除、状态筛选、阅读进度条、分页、标签系统），确保系统满足需求规格，保障整体质量。

## 2. 测试范围

### 2.1 测试模块与优先级

| 测试模块 | 测试内容 | 优先级 |
|----------|----------|--------|
| 图书管理 | 添加（含标签）、编辑（含标签）、单本删除 | 高 |
| 批量删除 | 勾选删除、全选删除、取消删除 | 高 |
| 搜索功能 | 按书名、作者模糊搜索 | 高 |
| 状态筛选 | 按阅读状态筛选，与搜索组合筛选 | 高 |

| 统计看板 | 总数、分类统计、状态统计正确性 | 中 |

## 3. 测试策略

### 3.1 测试方法
- **黑盒手工测试**：从用户视角操作界面，验证功能是否符合预期
- **边界值测试**：针对必填项、输入长度、特殊字符设计用例
- **组合测试**：验证搜索+状态+标签三条件组合筛选
- **回归测试**：所有功能开发完成后全面回归验证

### 3.2 测试环境

| 项目 | 配置 |
|------|------|
| 操作系统 | Windows 11 |
| 浏览器 | Chrome 最新版、Edge 最新版 |
| Python版本 | 3.12 |
| 依赖 | Flask |
| 测试数据 | 使用 `sql/schema.sql` 初始化的示例数据（含标签关联），并手工补充至少10条测试数据 |

### 4.2 测试数据准备
- 执行 `sql/schema.sql` 获得示例数据（3本图书+5个分类+5个标签+6条标签关联）
- 若从旧版数据库升级，运行 `python migrate_db.py` 新增标签表
- 手动添加以下测试数据：
  - 不同状态的图书各至少3本（未读、在读、已读）
  - 不同分类的图书，每个分类至少1本
  - 含多标签的图书（同一本关联2-3个标签）
  - 书名含特殊字符的图书（如 `C++ Primer`）
  - 书名/作者为英文的图书
  - 自定义标签的图书（如"2024年阅读"）
  - 总数超过10本以测试分页功能

## 6. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 测试数据不足 | 覆盖率低 | 提前规划并手工添加充分的数据，覆盖各种组合场景 |
| Bug修复引入新问题 | 回归不彻底 | 每次修复后执行关联用例，标签删除等涉及多表的操作重点回归 |
| 标签系统数据库迁移问题 | 旧数据无法使用新功能 | 提供 `migrate_db.py` 迁移脚本，测试前先执行迁移 |
| 时间紧张 | 测试不充分 | 优先执行高优先级用例，低风险用例可简化验证 |
