## 一、项目概述

### 1. 项目背景

随着个人藏书和电子资料不断增多，如何高效管理图书信息成为实际问题。现有的通用笔记或表格工具缺乏针对图书管理专门优化，无法满足分类检索、状态跟踪等需求。本项目旨在开发一个轻量级的个人图书管理助手，帮助用户便捷地记录、查找和整理图书。

### 2. 项目目标

- 实现图书信息的增删改查（CRUD）功能
- 支持按分类、书名、作者进行检索
- 提供图书数量统计与分类分布展示
- 提供简洁易用的 Web 界面，支持本地部署运行

### 3. 开发环境与条件

| 项目 | 内容 |
|------|------|
| 操作系统 | Windows |
| 开发语言 | Python 3.12 |
| Web 框架 | Flask |
| 数据库 | SQLite |
| 前端 | HTML + Jinja2 模板 + 原生 JavaScript |
| 开发工具 | VS Code |
| 版本控制 | Git + GitHub |

## 二、需求分析

### 1. 目标用户与使用场景

- 目标用户：个人用户、学生、有图书管理需求的读者
- 使用场景：个人电脑本地运行，用于管理纸质书、电子书或技术资料

### 2. 核心功能清单

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 图书列表 | 展示所有图书，支持表格显示 | 高 |
| 添加图书 | 录入书名、作者、ISBN、分类、位置、状态 | 高 |
| 编辑图书 | 修改已有图书信息 | 高 |
| 删除图书 | 删除不需要的图书 | 高 |
| 搜索功能 | 按书名或作者模糊搜索 | 中 |
| 分类管理 | 预设分类（Java/Python/数据库/前端/其他），可扩展 | 中 |
| 统计看板 | 显示总图书数、各分类图书数量 | 中 |

### 3. 非功能需求

- **性能要求**：页面加载与操作响应时间 < 2 秒
- **安全要求**：本地运行无外部网络暴露，使用参数化查询防止 SQL 注入
- **兼容性要求**：支持 Chrome/Edge 浏览器最新版本
- **可维护性**：代码结构清晰，文档完整，Git 提交记录规范

## 三、技术路线

### 1. 总体架构

采用三层架构设计：

- 表现层：HTML 模板 + 原生 JavaScript
- 业务逻辑层：Flask 路由和视图函数
- 数据访问层：sqlite3 模块 + 原生 SQL

### 2. 主要技术

| 层次 | 技术选型 | 理由 |
|------|----------|------|
| 后端框架 | Flask | 轻量、简单、路由清晰 |
| 数据库 | SQLite | 零配置、单文件、适合小型项目 |
| 前端模板 | Jinja2 | Flask 原生支持 |
| 样式 | 原生 CSS | 简单可控，无需复杂框架 |
| 版本控制 | Git + GitHub | 课程考核要求 |

### 3. 数据设计初步考虑

**主要实体：**
- 图书（Book）
- 分类（Category）

**实体关系：**
- 一个分类对应多本图书（一对多）

**关键数据表：**

| 表名 | 主要字段 |
|------|----------|
| category | id（主键），name（分类名） |
| book | id（主键），title，author，isbn，category_id（外键），location，status |

**状态字段取值：** 未读 / 在读 / 已读

数据将持久化存储在 SQLite 数据库文件中，通过参数化查询确保数据安全。
