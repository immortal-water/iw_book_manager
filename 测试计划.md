## 1. 测试目标

验证“个人图书管理助手”系统已实现功能的正确性、数据准确性和界面可用性，并确保新增扩展功能满足需求规格，最终保障系统整体质量。

---

## 2. 测试范围

### 2.1 测试模块与优先级

| 测试模块 | 测试内容 | 优先级 |
|----------|----------|--------|
| 图书管理 | 添加、编辑、删除图书 | 高 |
| 搜索功能 | 按书名、作者模糊搜索 | 高 |
| 统计看板 | 总数与分类数量正确性 | 中 |
| 分类关联 | 分类选择、联表显示 | 中 |
| UI交互 | 弹窗、按钮、表单提示 | 中 |
| 边界与异常 | 空输入、特殊字符、超长文本 | 中 |
| 扩展功能（计划） | CSV导出、批量删除、状态筛选等（开发后补充测试） | 中/低 |

### 2.2 不测试范围
- SQLite数据库引擎本身
- Flask框架核心功能
- 深度浏览器兼容性（仅验证 Chrome/Edge）

---

## 3. 测试策略

### 3.1 测试方法
- **黑盒手工测试**：从用户视角操作界面，验证功能是否符合预期
- **边界值测试**：针对必填项、输入长度、特殊字符设计用例
- **回归测试**：扩展功能新增后，复测原有核心功能

### 3.2 测试环境

| 项目 | 配置 |
|------|------|
| 操作系统 | Windows 11 |
| 浏览器 | Chrome 最新版、Edge 最新版 |
| Python版本 | 3.12 |
| 依赖 | Flask |
| 测试数据 | 使用 `sql/schema.sql` 初始化的3条示例数据，并手工补充至少10条测试数据 |
